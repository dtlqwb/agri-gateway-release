## 🔍 常见问题排查

### 问题1：看不到"🎯 作业类型"标签

**原因**：浏览器缓存  
**解决**：强制刷新页面（Ctrl+F5）或清除缓存

---

### 问题2：创建配置时提示"请先登录"

**原因**：Token过期  
**解决**：重新登录系统

---

### 问题3：预览显示"没有匹配的记录"

**原因**：
- 日期范围内没有数据
- 数据源选择错误
- 设备号或合作社名称不匹配

**解决**：
1. 检查数据库中有无符合条件的记录
2. 确认数据源是否正确（old_api / beidou）
3. 扩大日期范围测试

---

### 问题4：应用配置后数据未更新

**原因**：
- 配置条件太严格，没有匹配的记录
- 数据库中该字段已经是目标值

**解决**：
1. 先预览确认有匹配的记录
2. 检查数据管理页面中的实际数据
3. 查看控制台日志是否有错误

## 📊 数据库验证

可以直接查询数据库验证配置是否生效：

```sql
-- 查看配置表
SELECT * FROM work_type_config ORDER BY priority DESC;

-- 查看某段时间的作业类型分布
SELECT 
  work_type_name,
  COUNT(*) as record_count,
  SUM(acre) as total_acre
FROM work_records 
WHERE source = 'old_api'
  AND work_date BETWEEN '2026-03-01' AND '2026-05-31'
GROUP BY work_type_name;

-- 查看特定设备的作业类型
SELECT 
  work_date,
  t_number,
  org_name,
  work_type_name,
  acre
FROM work_records 
WHERE t_number = '386934'
  AND source = 'old_api'
ORDER BY work_date DESC
LIMIT 20;
```
