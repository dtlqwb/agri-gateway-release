# 农机定位数据聚合平台 - 项目完整文档

## 📋 项目概述

### 项目背景
本项目是为农业部门开发的农机作业数据聚合平台，主要解决以下问题：
1. 整合多个供应商（云途安等）的农机定位数据
2. 提供统一的数据管理和查询接口
3. 为农户提供作业面积查询服务
4. 解决数据重复、同步延迟等问题

### 技术架构
- **后端框架**: Node.js + Express
- **数据库**: MySQL 8.0 (从 SQLite 迁移)
- **前端**: 原生 HTML + JavaScript (无框架)
- **数据源**: 云途安 API + 旧供应商爬虫
- **部署**: 单服务器部署

### 核心功能
1. 多源数据同步与整合
2. 分级权限管理系统
3. 作业数据修复与编辑
4. 农户信息查询与统计
5. 数据导出功能

```javascript
// 基础认证中间件
function requireAuth(req, res, next) {
  const token = req.headers['authorization'];
  if (!token) {
    return res.status(401).json({ error: '未登录' });
  }
  
  // 验证token
  try {
    const user = verifyToken(token);
    if (!user) {
      return res.status(401).json({ error: '登录已过期' });
    }
    req.user = user;
    next();
  } catch (err) {
    return res.status(401).json({ error: '认证失败' });
  }
}

// 超级管理员中间件
function requireSuper(req, res, next) {
  requireAuth(req, res, () => {
    if (req.user.role !== 'super') {
      return res.status(403).json({ error: '仅超级管理员可操作' });
    }
    next();
  });
}
```

**问题7: 循环依赖导致服务启动失败**
```
错误信息:
ReferenceError: queryOne is not defined
    at Object.<anonymous> (services/workRecordsService.js:5:3)

问题分析:
1. db.js 使用展开语法引入 workRecordsService:

function getDbFunctions() {
  const db = require('./db');
  return {
    queryOne: db.queryOne,
    queryAll: db.queryAll,
    runSql: db.runSql
  };
}

async function getWorkRecords(filters = {}) {
  // 在函数调用时才获取数据库函数
  const { queryOne, queryAll } = getDbFunctions();
  // ... 使用 queryOne 和 queryAll

```

优势:
- 避免循环依赖
- 保持代码模块化
- 不影响原有架构

**问题8: MySQL 预处理语句 LIMIT 参数错误**
```
错误信息:
Incorrect arguments to mysqld_stmt_execute

问题原因:
MySQL 的预处理语句不支持将 LIMIT 和 OFFSET 的值作为参数传递

错误代码:
const records = await queryAll(
  `SELECT * FROM work_records LIMIT ? OFFSET ?`,
  [...params, pageSize, offset]  // pageSize 和 offset 是字符串
);

解决方案:
使用模板字符串直接嵌入整数值:
```javascript
const records = await queryAll(
  `SELECT * FROM work_records 
   LIMIT ${parseInt(pageSize)} OFFSET ${parseInt(offset)}`,
  params  // 只传递查询条件的参数
);
```

注意事项:
- 必须使用 parseInt() 转换为整数
- 防止 SQL 注入: pageSize 和 offset 来自前端，需要验证
- 这是 MySQL 驱动的限制，不是 SQL 语法问题
```

**问题9: 批量操作事务处理**
```
挑战:
批量更新/删除时需要保证数据一致性

解决方案:
使用 MySQL 事务:
```javascript
async function batchUpdateWorkRecords(ids, updates) {
  const conn = await pool.getConnection();
  try {
    await conn.beginTransaction();
    
    for (const id of ids) {
      // 记录修改历史
      await conn.execute(
        'INSERT INTO work_record_changes ...',
        [id, ...]
      );
      
      // 更新记录
      await conn.execute(
        'UPDATE work_records SET ... WHERE id = ?',
        [id, ...]
      );
    }
    
    await conn.commit();
  } catch (err) {
    await conn.rollback();
    throw err;
  } finally {
    conn.release();
  }
}
```

- 日期函数: datetime('now') vs NOW()
- 字符串连接: || vs CONCAT()
```

#### 1.2 索引优化
```sql
-- 初始索引（查询慢）

CREATE INDEX idx_org ON work_records(org_id);

-- 优化后（查询快 10 倍）

CREATE INDEX idx_source_date ON work_records(source, work_date);

优化原则:
- 复合索引按照查询频率排序
- 高基数字段放前面
- 覆盖查询条件的所有字段
```

### 2. 权限与安全

#### 2.1 Token 认证
```javascript
// Token 生成
function generateToken(user) {
  const payload = {
    id: user.id,
    username: user.username,
    role: user.role,
    exp: Date.now() + 7 * 24 * 60 * 60 * 1000  // 7天过期
  };
  return Buffer.from(JSON.stringify(payload)).toString('base64');
}

// Token 验证
function verifyToken(token) {
  try {
    const payload = JSON.parse(Buffer.from(token, 'base64').toString());
    if (payload.exp < Date.now()) {
      return null;  // 已过期
    }
    return payload;
  } catch (err) {
    return null;
  }
}

#### 4.2 数据同步容错
```javascript
// 同步失败自动重试
async function syncWithRetry(fn, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      return await fn();
    } catch (err) {
      console.error(`同步失败 (第${i+1}次尝试):`, err.message);
      
      if (i === retries - 1) {
        throw err;  // 最后一次失败，抛出异常
      }
      
      // 等待后重试（指数退避）
      const delay = Math.pow(2, i) * 1000;
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
