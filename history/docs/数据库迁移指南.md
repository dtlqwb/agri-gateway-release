# 数据库迁移到云端完整指南

## 📋 迁移前准备

### 1. 云端数据库要求

确保您的云端MySQL数据库满足以下条件：
- ✅ MySQL版本 >= 8.0
- ✅ 已创建数据库（如 `agri_gateway`）
- ✅ 有远程访问权限
- ✅ 防火墙已开放3306端口（或自定义端口）
- ✅ 有足够的存储空间

### 2. 需要的信息

从云服务提供商获取以下信息：
```bash
云端MySQL主机: your-cloud-mysql-host.com
端口: 3306（或自定义）
用户名: your_username
密码: your_password
数据库名: agri_gateway
```

## 🚀 迁移方法（3种方式）

### 方法1：使用mysqldump（推荐）⭐

#### 步骤1：导出本地数据库

打开命令行，执行：

```bash
# Windows PowerShell
mysqldump -u root -proot123 --single-transaction --routines --triggers agri_gateway > agri_gateway_backup.sql

# 或者 Linux/Mac
mysqldump -u root -p agri_gateway --single-transaction --routines --triggers > agri_gateway_backup.sql
```

**参数说明**：
- `--single-transaction`: 保证数据一致性
- `--routines`: 包含存储过程和函数
- `--triggers`: 包含触发器

#### 步骤2：传输SQL文件到云端

可以通过以下方式传输：
```bash
# 使用SCP（Linux/Mac）
scp agri_gateway_backup.sql user@cloud-server:/path/to/

# 使用FTP/SFTP工具（Windows）
# 推荐使用 FileZilla、WinSCP 等工具

# 或者通过云存储中转
# 上传到阿里云OSS、腾讯云COS等，然后在云端下载
```

#### 步骤3：导入到云端数据库

在云端服务器上执行：

```bash
# 先创建数据库（如果不存在）
mysql -h your-cloud-host -u your_username -p -e "CREATE DATABASE IF NOT EXISTS agri_gateway CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 导入数据
mysql -h your-cloud-host -u your_username -p agri_gateway < agri_gateway_backup.sql
```

### 方法2：使用Navicat等图形化工具（最简单）⭐⭐⭐

#### 步骤1：连接两个数据库

1. 打开Navicat Premium
2. 创建两个连接：
   - 本地连接：localhost
   - 云端连接：your-cloud-host

#### 步骤2：数据传输

1. 右键点击本地数据库 `agri_gateway`
2. 选择"数据传输"
3. 目标选择云端连接
4. 勾选所有表
5. 点击"开始"

**优点**：
- ✅ 可视化操作，简单直观
- ✅ 自动处理字符集和编码
- ✅ 可以预览和选择性传输
- ✅ 支持断点续传

---

### 方法3：使用Node.js脚本（自动化）

项目中已包含迁移脚本，可以直接使用。

#### 步骤1：配置云端数据库

编辑 `.env` 文件，添加云端配置：

```bash
# 云端MySQL配置
CLOUD_MYSQL_HOST=your-cloud-host.com
CLOUD_MYSQL_PORT=3306
CLOUD_MYSQL_USER=your_username
CLOUD_MYSQL_PASSWORD=your_password
CLOUD_MYSQL_DATABASE=agri_gateway
```

#### 步骤2：运行迁移脚本

```bash
node scripts/migrate-to-cloud.js
```

## 🔍 常见问题

### Q1: mysqldump命令找不到？

**A**: 需要安装MySQL客户端工具

**Windows**:
```bash
# 下载MySQL Community Server
# https://dev.mysql.com/downloads/mysql/
# 安装时选择"MySQL Server"和"MySQL Client"
```

**Linux**:
```bash
sudo apt-get install mysql-client  # Ubuntu/Debian
sudo yum install mysql             # CentOS/RHEL
```

### Q2: 连接云端数据库超时？

**A**: 检查以下几点：
1. 云端MySQL是否允许远程访问
2. 防火墙是否开放3306端口
3. 安全组规则是否正确配置
4. 网络连接是否正常

**测试连接**：
```bash
telnet your-cloud-host 3306
# 或
nc -zv your-cloud-host 3306
```

### Q3: 导入时出现字符集错误？

**A**: 确保使用正确的字符集

```bash
# 导出时指定字符集
mysqldump -u root -p --default-character-set=utf8mb4 agri_gateway > backup.sql

# 导入时指定字符集
mysql -h cloud-host -u user -p --default-character-set=utf8mb4 agri_gateway < backup.sql
```

### Q4: 数据量很大，传输很慢？

**A**: 优化方法：
1. 压缩SQL文件
```bash
# 导出时压缩
mysqldump -u root -p agri_gateway | gzip > backup.sql.gz

# 传输压缩文件
scp backup.sql.gz user@cloud:/path/

# 云端解压并导入
gunzip < backup.sql.gz | mysql -h cloud-host -u user -p agri_gateway
```

2. 使用更快的传输方式（内网传输、专线等）

### Q5: 迁移后定时任务不执行？

**A**: 检查：
1. 服务器时间是否正确
2. node-cron是否正常运行
3. 查看日志是否有错误

## 🔐 安全建议

1. **使用SSL连接**（如果云端支持）
```bash
# .env中添加
MYSQL_SSL=true
```

2. **限制IP访问**
- 只允许特定IP访问云端数据库
- 使用白名单机制

3. **定期备份**
```bash
# 设置定时备份（每天凌晨3点）
0 3 * * * mysqldump -h cloud-host -u user -p agri_gateway | gzip > /backup/db_$(date +\%Y\%m\%d).sql.gz
```

4. **监控数据库性能**
- CPU使用率
- 内存使用
- 连接数
- 慢查询日志
