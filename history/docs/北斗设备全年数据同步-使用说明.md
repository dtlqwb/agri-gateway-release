# 北斗设备全年数据同步脚本 - 使用说明

## 📋 功能说明

该脚本用于从北斗API（旧供应商API）批量同步指定年份的所有日期数据。

**特点**：
- ✅ 自动跳过已有数据的日期（避免重复）
- ✅ 显示实时进度和统计信息
- ✅ 支持中断后继续执行
- ✅ 详细的日志输出
- ✅ 符合代码可维护性规范

---

## 🚀 使用方法

### 1. 前置条件

确保已完成以下配置：

#### .env 文件配置
```env
# 启用旧供应商API同步
ENABLE_OLD_SYNC=true

# API配置
OLD_API_BASE=http://60.188.243.23:28111
OLD_API_USER=your_user
OLD_API_KEY=your_key

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=agri_gateway
```

#### 导入设备映射表
如果还没有导入设备，先运行：
```bash
node scripts/import-old-devices.js
```

### 2. 执行同步

#### 同步今年（2026年）的数据
```bash
node scripts/sync-beidou-yearly.js
```

#### 同步指定年份的数据
```bash
node scripts/sync-beidou-yearly.js 2025
```

#### 同步其他年份
```bash
node scripts/sync-beidou-yearly.js 2024
```

## ⚙️ 工作原理

### 1. 检查环境
- 验证数据库连接
- 检查同步功能是否启用
- 确认设备列表不为空

### 2. 生成日期列表
- 计算指定年份的起止日期
- 生成所有日期的数组（365或366天）

### 3. 逐日同步
对于每个日期：
1. **检查是否已有数据**
   - 查询 `work_records` 表
   - 如果该日期已有记录，跳过
   
2. **调用API同步**
   - 遍历所有设备
   - 调用 `getAreaData(macid, date)` 获取面积
   - 保存到 `work_records` 表
   
3. **延迟处理**
   - 每次请求后延迟200ms
   - 避免API限流

### 4. 统计汇总
- 成功/失败天数
- 成功/失败次数
- 总作业面积

## ⚠️ 注意事项

### 1. 时间估算

**同步速度**：
- 每台设备约 0.3 秒（含延迟）
- 5台设备 × 365天 ≈ 9分钟
- 10台设备 × 365天 ≈ 18分钟

**实际时间可能更长**，取决于：
- API响应速度
- 网络状况
- 设备数量

### 2. API限流

脚本已内置200ms延迟，但如果遇到限流：
- 可以增加延迟时间（修改 `sleep(200)` 为更大的值）
- 或者分多次执行（例如按月同步）

### 3. 数据去重

脚本会自动检测并跳过已有数据的日期，判断依据：
```sql
SELECT COUNT(*) FROM work_records 
WHERE source = 'old_api' AND work_date = ?
```

如果该日期已有任何记录，整日跳过。

### 4. 错误处理

如果某天同步失败：
- 不会中断整个流程
- 会继续同步下一天
- 最终统计中会显示失败次数

## 📝 故障排查

### 问题1：提示"功能未启用"

**原因**：`.env` 文件中未设置 `ENABLE_OLD_SYNC=true`

**解决**：
```env
ENABLE_OLD_SYNC=true
```
然后重启服务。

### 问题2：提示"没有设备数据"

**原因**：未导入设备映射表

**解决**：
```bash
node scripts/import-old-devices.js
```

### 问题3：大量失败

**可能原因**：
- API地址不正确
- 认证信息错误
- 网络连接问题
- API服务不可用

**排查步骤**：
1. 检查 `.env` 中的API配置
2. 手动测试API：
   ```bash
   curl -X POST http://60.188.243.23:28111/machine/gps/area \
     -H "Content-Type: application/json" \
     -H "user: YOUR_USER" \
     -H "timestamp: $(date +%s)" \
     -H "sign: YOUR_SIGN" \
     -d '{"macid":"386934","day":"2026-01-01"}'
   ```

### 问题4：同步太慢

**优化方案**：
1. 减少延迟时间（有风险）：
   ```javascript
   await sleep(100); // 从200改为100
   ```

2. 并行处理（需要修改代码）：
   - 同时同步多个日期
   - 注意API限流

3. 分批次执行：
   - 先同步1-3月
   - 再同步4-6月
   - 依此类推
