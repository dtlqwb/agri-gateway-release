# 代码可维护性规范 - 实施指南

## 📋 规范要求

### 1. 注释规范 ✅

**要求**：每个函数必须有JSDoc注释，说明参数、返回值、功能

**标准格式**：
```javascript
/**
 * 函数简要描述
 * @param {类型} 参数名 - 参数说明
 * @param {类型} [可选参数] - 可选参数说明
 * @returns {类型} 返回值说明
 * @throws {Error} 可能抛出的错误
 * @description 详细的功能描述
 * @example 使用示例（可选）
 */
async function example(param1, param2) {
  // 实现
}
```

**改进前**：
```javascript
// 更新设备的作业类型
async function updateDeviceWorkType(macid, workTypeName) {
  await runSql(`UPDATE ...`, [workTypeName, macid]);
}
```

**改进后**：
```javascript
/**
 * 更新设备的作业类型
 * @param {string} macid - 设备终端编号
 * @param {string} workTypeName - 作业类型名称（如"玉米播种"）
 * @returns {Promise<void>}
 * @throws {Error} 当数据库更新失败时抛出错误
 * @description 更新指定设备的当前作业类型，影响后续数据同步时的作业类型设置
 * @example
 * await updateDeviceWorkType('386934', '玉米播种');
 */
async function updateDeviceWorkType(macid, workTypeName) {
  console.log(`[设备管理] 开始更新作业类型: macid=${macid}, type=${workTypeName}`);
  
  try {
    await runSql(`
      UPDATE old_supplier_devices 
      SET work_type_name = ?, updated_at = NOW()
      WHERE macid = ?
    `, [workTypeName, macid]);
    
    console.log(`[设备管理] 作业类型更新成功: macid=${macid}`);
  } catch (error) {
    console.error(`[设备管理] 作业类型更新失败: macid=${macid}, 错误: ${error.message}`);
    throw new Error(`更新设备作业类型失败: ${error.message}`);
  }
}
```

### 2. 日志完善 ✅

**要求**：关键步骤必须有日志（开始、结束、错误）

**日志级别**：
- `console.log()` - 正常流程（开始、成功）
- `console.warn()` - 警告信息（非致命问题）
- `console.error()` - 错误信息（异常情况）

**标准格式**：
```javascript
async function processData(data) {
  console.log(`[模块名] 开始处理: 参数=${JSON.stringify(keyParams)}`);
  
  try {
    // 处理逻辑
    console.log(`[模块名] 处理成功: 结果=${result}`);
    return result;
  } catch (error) {
    console.error(`[模块名] 处理失败: 错误=${error.message}`, error.stack);
    throw error;
  }
}
```

**关键位置必须加日志**：
1. 函数入口（开始）
2. API调用前后
3. 数据库操作前后
4. 文件读写前后
5. 异常捕获

### 3. 错误处理 ✅

**要求**：每个异步操作必须有try-catch，错误信息要具体

**标准模式**：
```javascript
async function doSomething(param) {
  try {
    // 业务逻辑
    const result = await someAsyncOperation(param);
    return result;
  } catch (error) {
    // 记录详细错误信息
    console.error(`[模块] 操作失败: param=${param}, 错误: ${error.message}`);
    
    // 抛出具体的错误
    throw new Error(`具体操作失败: ${error.message}`);
  }
}
```

**错误分类处理**：
```javascript
try {
  await apiCall();
} catch (error) {
  if (error.response) {
    // HTTP错误
    console.error(`API错误: ${error.response.status} ${error.response.statusText}`);
  } else if (error.code === 'ECONNREFUSED') {
    // 连接错误
    console.error('无法连接到服务器');
  } else {
    // 其他错误
    console.error(`未知错误: ${error.message}`);
  }
  throw error;
}
```

### 5. 单一职责 ⚠️

**要求**：一个函数只做一件事，超过30行必须拆分

**改进前**（❌ 函数过长）：
```javascript
async function syncData() {
  // 100行代码...
  // - 获取设备列表
  // - 遍历设备
  // - 调用API
  // - 解析数据
  // - 保存到数据库
  // - 记录日志
}
```

**改进后**（✅ 拆分为小函数）：
```javascript
/**
 * 同步数据主流程
 */
async function syncData() {
  console.log('[同步] 开始数据同步');
  
  const devices = await getDeviceList();
  const results = await processDevices(devices);
  await saveResults(results);
  
  console.log('[同步] 数据同步完成');
}

/**
 * 获取设备列表
 */
async function getDeviceList() {
  // 不超过30行
}

/**
 * 处理设备数据
 */
async function processDevices(devices) {
  // 不超过30行
}

/**
 * 保存同步结果
 */
async function saveResults(results) {
  // 不超过30行
}
```

**拆分原则**：
1. 每个函数只做一件事
2. 函数名清晰表达功能
3. 函数长度控制在30行以内
4. 提取重复代码为独立函数
5. 复杂逻辑拆分为多个步骤

### 6. 测试友好 ⚠️

**要求**：核心逻辑不依赖外部状态，方便单元测试

**改进前**（❌ 依赖全局状态）：
```javascript
async function calculateAcre(data) {
  // 直接读取process.env
  const rate = process.env.ACRE_RATE || 1.0;
  return data.rawAcre * rate;
}
```

**改进后**（✅ 参数化，易测试）：
```javascript
/**
 * 计算作业面积
 * @param {Object} data - 原始数据
 * @param {number} [rate=1.0] - 转换系数
 * @returns {number} 计算后的面积
 */
function calculateAcre(data, rate = 1.0) {
  return data.rawAcre * rate;
}

// 使用时
const { BUSINESS_CONFIG } = require('../config');
const acre = calculateAcre(data, BUSINESS_CONFIG.ACRE_CONVERSION_RATE);
```

**测试友好设计原则**：
1. 纯函数优先（无副作用）
2. 依赖注入（通过参数传入）
3. 避免全局状态
4. 返回明确的结果
5. 便于mock外部依赖

## 💡 最佳实践示例

### 示例1：完整的函数模板

```javascript
/**
 * 函数功能简述
 * @param {类型} param1 - 参数1说明
 * @param {类型} param2 - 参数2说明
 * @returns {Promise<类型>} 返回值说明
 * @throws {Error} 错误说明
 * @description 详细的功能描述和使用场景
 * @example
 * const result = await functionName('value1', 'value2');
 */
async function functionName(param1, param2) {
  console.log(`[模块名] 开始执行: param1=${param1}, param2=${param2}`);
  
  try {
    // 参数验证
    if (!param1) {
      throw new Error('param1不能为空');
    }
    
    // 业务逻辑
    const result = await doSomething(param1, param2);
    
    console.log(`[模块名] 执行成功: result=${JSON.stringify(result)}`);
    return result;
  } catch (error) {
    console.error(`[模块名] 执行失败: param1=${param1}, 错误: ${error.message}`, error.stack);
    throw new Error(`函数functionName执行失败: ${error.message}`);
  }
}
```
