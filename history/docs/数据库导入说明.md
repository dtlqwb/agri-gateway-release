## ⚙️ 导入前准备

### 1. 确保MySQL已安装

检查MySQL是否运行：
```bash
# Windows
services.msc  # 查看MySQL服务是否启动

# Linux
systemctl status mysql

# Mac
brew services list | grep mysql
```

### 2. 创建数据库（可选）

SQL文件中已包含创建数据库的命令，但如果想手动创建：

```sql
CREATE DATABASE IF NOT EXISTS agri_gateway 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

### 3. 检查权限

确保MySQL用户有以下权限：
- CREATE（创建数据库和表）
- INSERT（插入数据）
- SELECT（查询数据）
- DROP（删除表，用于重建）

## ❓ 常见问题

### Q1: 导入时提示"Access denied"？

**A**: 权限不足，请检查：
1. MySQL用户名和密码是否正确
2. 用户是否有CREATE权限
3. 尝试使用root用户导入

### Q2: 提示"Unknown database"？

**A**: SQL文件中已包含创建数据库的命令，如果仍然报错：
1. 手动创建数据库
2. 或者去掉SQL文件中的 `CREATE DATABASE` 语句

### Q3: 导入速度很慢？

**A**: 优化方法：
```sql
-- 导入前执行
SET GLOBAL innodb_flush_log_at_trx_commit = 0;
SET GLOBAL sync_binlog = 0;

-- 导入后恢复
SET GLOBAL innodb_flush_log_at_trx_commit = 1;
SET GLOBAL sync_binlog = 1;
```

### Q4: 出现字符集乱码？

**A**: 确保使用正确的字符集：
```bash
mysql -u user -p --default-character-set=utf8mb4 agri_gateway < backup.sql
```

### Q5: 导入中断怎么办？

**A**: 
1. 删除已导入的数据
2. 重新导入
3. 或者使用Navicat，支持断点续传
