- ⏰ **定时任务**: 每天凌晨 2:00 自动执行

#### 旧供应商API（需手动触发或等待定时）

- 🔄 **同步策略**: 只同步**昨天**的数据
- 📅 **最新数据**: 2026-04-14（前天）

### 3. 为什么旧供应商数据只到14日？

**可能原因：**

1. **CSV导入的历史数据只到13日**
   - 之前通过 `scripts/import-old-devices.js` 导入的是静态CSV文件
   - CSV中的数据可能是截止到4月13日的历史数据

2. **定时同步还未执行今天的任务**
   - 定时任务在凌晨4点执行，同步的是"昨天"的数据
   - 今天是4月17日，凌晨4点应该同步4月16日的数据
   - 但实际只看到4月14日的数据，说明**4月16日和17日凌晨的同步可能失败了**

3. **API调用失败或无数据**
   - 旧供应商API可能在某些日期没有返回数据
   - 或者API调用过程中出现错误

## 💡 解决方案

### 方案1：手动触发同步（推荐立即执行）

创建手动同步脚本，立即同步缺失的日期：

```javascript
// scripts/manual-sync-old.js
require('dotenv').config();
const oldService = require('../services/oldSupplierService');

(async () => {
  // 同步4月15日、16日的数据
  const dates = ['2026-04-15', '2026-04-16'];
  
  for (const date of dates) {
    console.log(`\n========== 同步 ${date} ==========`);
    const result = await oldService.syncDayData(date);
    console.log('结果:', result);
  }
})();
```

**执行命令：**
```bash
node scripts/manual-sync-old.js
```

### 方案2：检查定时任务是否正常运行

查看服务日志，确认定时任务是否正常执行：

```bash
# 查看最近的日志

# 或者检查数据库中的同步记录
```

### 方案3：调整同步策略

如果希望旧供应商数据更及时，可以：

1. **增加同步频率**：从每天1次改为每6小时1次
2. **扩大同步范围**：从只同步昨天改为同步最近3天
3. **添加失败重试机制**：同步失败后自动重试

## 📝 技术细节

### 旧供应商API同步流程

```
定时任务触发（凌晨4:00）
    ↓
读取 old_supplier_devices 表（73台设备）
    ↓
逐个调用 API: /machine/gps/area
    ↓
解析响应: { code: 200, data: "2.6793" }
    ↓
保存到 work_records 表
    ↓
更新统计信息
```

### 关键配置

**.env 文件：**
```env
# 启用旧供应商API同步
ENABLE_OLD_SYNC=true

# 旧供应商API配置
OLD_SUPPLIER_BASE_URL=http://xxx.xxx.xxx.xxx/api
OLD_SUPPLIER_USER=your_user
OLD_SUPPLIER_KEY=your_key
```

**scheduler.js：**
```javascript
// 每天凌晨 4:00 自动同步
next.setHours(4, 0, 0, 0);

const result = await oldService.syncDayData();
```
