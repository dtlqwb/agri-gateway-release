# 云途安手动同步功能修复说明

## 🐛 问题描述

用户在点击"手动同步"按钮时，出现错误提示。

**原因分析**：
- 前端发送的参数是 `{ date }`（日期选择器的值）
- 后端期望的参数是 `{ mode }`（同步模式：'all' / 'recent'）
- 参数不匹配导致后端无法正确识别同步类型

---

## ✅ 修复方案

### 1. 前端修改 (`public/index.html`)

**修改前**：
```javascript
body: JSON.stringify({ date })
```

**修改后**：
```javascript
// 确定同步模式
let mode = 'recent'; // 默认增量同步（最近7天）
if (date) {
  // 如果选择了具体日期，同步指定日期
  mode = 'single';
}

body: JSON.stringify({ mode, date })
```

**改进点**：
- ✅ 不再强制要求选择日期
- ✅ 未选择日期时，默认同步最近7天数据
- ✅ 选择日期后，同步指定日期的数据
- ✅ 支持三种模式：全量、增量、指定日期

---

### 2. 后端修改 (`index.js`)

**修改前**：
```javascript
const { mode } = req.body;
const syncMode = (mode === 'all') ? 'all' : 'recent';
```

**修改后**：
```javascript
const { mode, date } = req.body;
let syncMode = 'recent';
let syncDate = new Date().toISOString().substring(0, 10);

if (mode === 'all') {
  syncMode = 'all';
} else if (mode === 'single' && date) {
  syncMode = 'single';
  syncDate = date;
}
```

**改进点**：
- ✅ 支持 `single` 模式（同步指定日期）
- ✅ 根据用户选择的日期动态设置同步目标
- ✅ 日志和响应中显示正确的同步类型

---

## 🎯 使用方式

### 方式1：增量同步（推荐）⭐

**操作**：不选择日期，直接点击"手动同步"

**效果**：
- 同步最近7天的数据
- 快速完成，适合日常更新

**适用场景**：
- 每天定期同步
- 检查是否有遗漏数据

---

### 方式2：指定日期同步

**操作**：
1. 在日期选择器中选择具体日期（如 2026-04-10）
2. 点击"手动同步"

**效果**：
- 只同步该日期的数据
- 精确控制同步范围

**适用场景**：
- 补录历史数据
- 重新同步某一天的数据
- 排查特定日期的问题

---

### 方式3：全量同步

**操作**：通过API调用
```bash
curl -X POST http://localhost:3001/api/sync/yuntinan \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"mode": "all"}'
```

**效果**：
- 同步所有历史数据
- 耗时较长

**适用场景**：
- 首次部署
- 数据严重不一致时重建

---

## 📊 同步模式对比

| 模式 | 同步范围 | 速度 | 适用场景 |
|------|---------|------|---------|
| **增量** (recent) | 最近7天 | ⚡ 快 | 日常更新 |
| **指定日期** (single) | 选定的一天 | ⚡⚡ 最快 | 补录/重同步 |
| **全量** (all) | 所有历史数据 | 🐢 慢 | 首次部署/重建 |
