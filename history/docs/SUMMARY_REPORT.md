### 3. Bug检查与修复

#### Bug #1: UTC时间显示问题 ✅ 已修复
**严重程度**: 🔴 高  
**影响范围**: 所有显示日期的地方  

**问题**: 
- 数据库存储UTC时间 `2026-04-16T16:00:00.000Z`
- 前端直接显示，用户误以为是4月16日
- 实际是北京时间4月17日 00:00:00

**修复**:
```javascript
// services/workRecordsService.js
// 将UTC时间转换为本地日期格式（YYYY-MM-DD）
records.forEach(record => {
  if (record.work_date) {
    const date = new Date(record.work_date);
    const year = date.getFullYear();
    const month = String(date.getMonth() + 1).padStart(2, '0');
    const day = String(date.getDate()).padStart(2, '0');
    record.work_date = `${year}-${month}-${day}`;
  }
});
```

**验证**: ✅ 已通过测试

#### Bug #3: 旧供应商数据源筛选无结果 ✅ 已修复
**严重程度**: 🔴 高  
**影响范围**: 数据修复页面筛选、数据导出  

**问题**: 
- 前端传递 `source='old'`
- 数据库标识为 `'old_api'`
- SQL精确匹配找不到数据

**修复**:
```javascript
// services/workRecordsService.js
if (filters.source === 'old') {
  whereClause += ' AND source IN (?, ?)';
  params.push('old', 'old_api');
} else {
  whereClause += ' AND source = ?';
  params.push(filters.source);
}
```
