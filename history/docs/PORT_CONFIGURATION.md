# ⚠️ 端口号配置说明

**问题**: 你看到的端口号是 3000，但应该是 3001

---

## 🔍 原因分析

项目代码中的默认端口是 **3000**，但如果配置了 `.env` 文件中的 `PORT` 变量，会使用配置的端口（推荐 3001）。

你可能遇到了以下情况之一：

### 情况1: 没有创建 .env 文件
如果你直接运行 `node index.js`，但没有复制 `.env.example` 为 `.env`，系统会使用默认端口 3000。

### 情况2: .env 文件中没有配置 PORT
如果你的 `.env` 文件中没有 `PORT=3001` 这一行，也会使用默认端口 3000。

---

## ✅ 解决方案

### 方案1: 修改为 3001 端口（推荐）

#### 步骤1: 检查是否有 .env 文件
在项目根目录查看是否有 `.env` 文件。

如果没有，复制示例文件：
```bash
# Windows PowerShell
Copy-Item .env.example .env

# Linux/Mac
cp .env.example .env
```

#### 步骤2: 编辑 .env 文件
打开 `.env` 文件，找到这一行：
```env
PORT=3001
```

确保它没有被注释掉（前面没有 `#`）。

#### 步骤3: 重启服务
```bash
# 停止当前运行的服务（Ctrl+C）
# 然后重新启动
node index.js
```

现在应该看到：
```
🚀 聚合平台已启动: http://localhost:3001
```

---

### 方案2: 继续使用 3000 端口（也可以）

如果你觉得 3000 端口没问题，可以继续使用，不需要修改。

只需访问时使用正确的端口：
```
http://localhost:3000
```

**注意**: 如果选择这个方案，所有文档中提到的 3001 都要改为 3000。

---

### 方案3: 使用其他端口

如果你想用其他端口（比如 8080），在 `.env` 文件中修改：
```env
PORT=8080
```

然后重启服务即可。

---

## 📝 验证方法

### 1. 查看启动日志
运行 `node index.js` 后，看控制台输出：
```
🚀 聚合平台已启动: http://localhost:XXXX
```
这里的 XXXX 就是实际使用的端口号。

### 2. 访问测试
在浏览器中访问：
```
http://localhost:3001  （如果配置了3001）
或
http://localhost:3000  （如果使用默认3000）
```

能看到登录页面即成功。

### 3. API健康检查
```bash
curl http://localhost:3001/api/health
# 或
curl http://localhost:3000/api/health
```

应该返回：
```json
{"code":0,"msg":"ok","time":"2026-04-18T00:00:00.000Z"}
```

---

## 💡 建议

**推荐使用 3001 端口**，原因：
1. ✅ 避免与 Node.js 默认端口 3000 冲突
2. ✅ 与项目文档保持一致
3. ✅ 便于区分不同环境（开发3001、测试3002等）

---

## 🔧 如果端口被占用

如果启动时提示端口被占用：

### Windows
```powershell
# 查看哪个进程占用了端口
netstat -ano | findstr :3001

# 结束进程（替换 PID 为实际的进程ID）
taskkill /PID <PID> /F
```

### Linux/Mac
```bash
# 查看占用端口的进程
lsof -i :3001

# 结束进程
kill -9 <PID>
```

或者直接在 `.env` 中换一个端口：
```env
PORT=3002
```
