## 🎯 优势对比

### Playwright爬虫方式（已禁用）

**缺点**：
- ❌ 需要安装浏览器（Chrome/Chromium）
- ❌ 依赖网页结构，容易失效
- ❌ 速度慢，资源占用高
- ❌ 需要处理登录、验证码等
- ❌ 维护成本高

**优点**：
- ✅ 可以获取更详细的数据

### 新API方式（推荐使用）

**优点**：
- ✅ 无需浏览器，轻量级
- ✅ 响应速度快
- ✅ 稳定性高
- ✅ 维护简单
- ✅ 资源占用低

**缺点**：
- ⚠️ 依赖第三方API可用性

## 🔧 如何重新启用爬虫（如需要）

如果新API不可用，可以临时切换回爬虫方式：

### 步骤1：修改配置

编辑 `.env` 文件：

```bash
# 启用旧供应商爬虫
ENABLE_OLD_CRAWLER=true

# 可选：禁用新API
ENABLE_OLD_SYNC=false
```

### 步骤2：重启服务

```bash
npm start
```

### 步骤3：验证

查看启动日志：
```
[旧供应商抓取] 下次定时抓取: 2026/4/13 03:00:00 (XXX 分钟后)
```

## ❓ 常见问题

### Q1: 为什么要禁用爬虫？

**A**: 
- 新API更稳定、更高效
- 减少系统复杂度
- 降低维护成本
- 节省服务器资源

### Q2: 如果新API挂了怎么办？

**A**: 
1. 临时启用爬虫（ENABLE_OLD_CRAWLER=true）
2. 使用Excel手工导入
3. 联系技术支持

### Q3: 历史爬虫数据会丢失吗？

**A**: 
不会。历史数据仍然保留在数据库中，只是不再自动抓取新数据。

### Q4: 可以同时使用两种方式吗？

**A**: 
技术上可以，但不建议。会导致：
- 数据重复
- 资源浪费
- 管理混乱

建议使用单一数据源。
