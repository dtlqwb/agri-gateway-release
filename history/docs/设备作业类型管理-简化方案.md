# 设备作业类型管理 - 简化方案

## 📋 核心思路

1. **统一数据源**：北斗设备 = 旧供应商API，都使用 `source='old_api'`
2. **作业类型存储在设备表中**：在 `old_supplier_devices.work_type_name` 字段
3. **农户端修改**：农户可以在 farmer.html 页面修改设备的作业类型
4. **自动应用**：同步数据时读取设备的当前作业类型

---

## ✅ 已完成的功能

### 1. 数据库表结构（services/db.js）

```sql
CREATE TABLE IF NOT EXISTS old_supplier_devices (
  id INT AUTO_INCREMENT PRIMARY KEY,
  macid VARCHAR(20) NOT NULL UNIQUE COMMENT '终端编号',
  cooperative_name VARCHAR(100) DEFAULT '' COMMENT '合作社名称',
  driver_name VARCHAR(50) DEFAULT '' COMMENT '机手姓名',
  work_type_name VARCHAR(100) DEFAULT '其他' COMMENT '当前作业类型',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**关键字段**：`work_type_name` - 存储设备的当前作业类型

---

```javascript
batchUpdateDeviceWorkType(macids, workTypeName)
```

---

### 3. API接口（index.js）

#### ① 农户修改设备作业类型
```
POST /api/device/work-type
Body: { macid, workTypeName }
权限：无需登录（农户端直接调用）
```

#### ② 获取设备列表
```
GET /api/devices/list?orgId=xxx
返回：设备列表（含作业类型）
```

#### ③ 批量更新设备作业类型
```
POST /api/device/work-type/batch
Body: { macids: [], workTypeName }
权限：超级管理员
```

---

### 4. 数据同步逻辑（services/oldSupplierService.js）

同步数据时，自动读取设备的当前作业类型：

```javascript
  const workTypeName = await db.getDeviceWorkType(device.macid);
  
  // 插入/更新记录时使用该作业类型
  INSERT INTO work_records (..., work_type_name, ...) 
  VALUES (..., ?, ...)
}
```

## 🔧 使用示例

### 示例1：农户修改单台设备

```bash
curl -X POST http://localhost:3001/api/device/work-type \
  -H "Content-Type: application/json" \
  -d '{"macid":"386934","workTypeName":"玉米播种"}'
```

**结果**：
- 设备386934的 `work_type_name` 更新为"玉米播种"
- 下次同步的数据都会使用"玉米播种"

## ⚠️ 注意事项

### 1. 历史数据不会自动更新
- 修改设备的作业类型后，**只影响未来的同步数据**
- 历史数据需要使用数据管理页面手动修改

### 2. 默认值
- 新导入的设备，`work_type_name` 默认为"其他"
- 建议农户登录后先设置作业类型

### 3. 数据来源统一
- 北斗设备和旧供应商API是同一个数据源
- 都使用 `source='old_api'`
- 不需要区分

## 📊 与之前方案的对比

| 特性 | 旧方案（按日期配置） | 新方案（设备级别） |
|------|-------------------|------------------|
| 配置粒度 | 全局/合作社/终端 + 日期范围 | 仅终端级别 |
| 灵活性 | 高（可按时间段配置） | 中（只能整体修改） |
| 复杂度 | 高（需要配置表） | 低（只需设备表） |
| 易用性 | 复杂（多个配置项） | 简单（直接修改） |
| 适用场景 | 季节性变化频繁 | 作业类型相对稳定 |

**结论**：新方案更简单实用，符合农户的使用习惯。
