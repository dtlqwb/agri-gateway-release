> 本文档供王会强参考，用于开发Java版本后端
> 基于Node.js版本功能梳理，保留核心设计，优化工程实现

## 一、项目概述

### 1.1 项目背景
- **业务目标**：为灵丘县农业局提供农机作业监管平台，实现补贴发放依据的数据支撑
- **核心流程**：GPS轨迹 → 作业面积计算 → 农户确认 → 补贴核算
- **双供应商**：
  - **云途安（新供应商）**：42台设备，JWT认证，API较完善
  - **旧供应商（北斗农机）**：73台设备，MD5签名认证，API较简陋

### 1.2 技术栈建议
| 层级 | 推荐方案 | 说明 |
|------|----------|------|
| 后端框架 | Spring Boot 3.x | 成熟稳定，生态丰富 |
| 数据库 | MySQL 8.x | 兼容现有数据结构 |
| 缓存 | Redis | 会话、热点数据缓存 |
| ORM | MyBatis-Plus | 简化CRUD，支持分页 |
| 定时任务 | XXL-Job / Quartz | 分布式任务调度 |
| 日志 | SLF4J + Logback | 分级日志，便于排查 |
| 安全 | Spring Security + JWT | 统一认证授权 |

### 2.2 表结构详细定义

#### 2.2.1 organizations（合作社/公司表）

```sql
CREATE TABLE organizations (
  id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
  name VARCHAR(255) NOT NULL COMMENT '组织名称（全称）',
  short_name VARCHAR(100) DEFAULT '' COMMENT '简称',
  machine_count INT DEFAULT 0 COMMENT '农机数量（冗余统计）',
  total_acre DECIMAL(10,2) DEFAULT 0 COMMENT '累计作业面积（亩）',
  source VARCHAR(20) DEFAULT 'yuntinan' COMMENT '数据来源：yuntinan-云途安, old-旧供应商, manual-手动添加',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_source (source)
) COMMENT='合作社/公司表';
```

#### 2.2.2 machines（农机设备表）

```sql

  year_acre DECIMAL(10,2) DEFAULT 0 COMMENT '年度作业面积（亩）',

  source VARCHAR(20) DEFAULT 'yuntinan' COMMENT '数据来源',
  last_sync_at TIMESTAMP NULL COMMENT '最后同步时间',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_source (source)

```

#### 2.2.3 work_records（作业记录表）★核心表

```sql
CREATE TABLE work_records (
  id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
  t_number VARCHAR(50) NOT NULL COMMENT '终端号',
  work_date DATE NOT NULL COMMENT '作业日期',
  work_type VARCHAR(20) DEFAULT '' COMMENT '作业类型编码',
  work_type_name VARCHAR(50) DEFAULT '' COMMENT '作业类型名称（如：旋耕、深翻）',
  
  -- 面积数据（核心字段）
  acre DECIMAL(10,4) DEFAULT 0 COMMENT '作业面积（亩）- API原始值',
  ok_acre DECIMAL(10,4) DEFAULT 0 COMMENT '达标面积（亩）- 用于补贴计算',
  repeat_acre DECIMAL(10,4) DEFAULT 0 COMMENT '重复面积（亩）',
  leave_acre DECIMAL(10,4) DEFAULT 0 COMMENT '漏耕面积（亩）',
  
  -- 位置信息
  slng DECIMAL(12,8) DEFAULT NULL COMMENT '起始经度',
  slat DECIMAL(12,8) DEFAULT NULL COMMENT '起始纬度',
  land_group JSON COMMENT '地块围栏坐标（GeoJSON格式）',
  
  -- 作业信息
  work_time INT DEFAULT 0 COMMENT '作业时长（分钟）',
  pass_count INT DEFAULT 0 COMMENT '作业遍数',
  avg_depth DECIMAL(5,2) DEFAULT NULL COMMENT '平均耕深（cm）',
  
  -- 数据来源
  source VARCHAR(20) DEFAULT 'yuntinan' COMMENT '数据来源：yuntinan, old, old_api, import',
  source_id VARCHAR(100) DEFAULT '' COMMENT '源系统记录ID（用于去重）',
  
  -- 确认状态（农户确认流程）
  confirm_status TINYINT DEFAULT 0 COMMENT '确认状态：0-未确认，1-已确认，2-有异议',
  confirm_by BIGINT COMMENT '确认人（农户ID）',
  confirm_at TIMESTAMP NULL COMMENT '确认时间',
  confirm_remark VARCHAR(500) DEFAULT '' COMMENT '确认备注',
  
  -- 补贴相关
  subsidy_rate DECIMAL(8,2) DEFAULT NULL COMMENT '补贴标准（元/亩）',
  subsidy_amount DECIMAL(10,2) DEFAULT NULL COMMENT '补贴金额（元）',
  
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  -- 索引
  UNIQUE KEY uk_record (t_number, work_date, work_type_name, source),
  INDEX idx_t_number (t_number),
  INDEX idx_work_date (work_date),
  INDEX idx_source (source),
  INDEX idx_confirm_status (confirm_status)
) COMMENT='作业记录表';
```

## 三、API接口清单

### 3.1 接口设计规范

- **基础路径**：`/api`
- **响应格式**：统一JSON
  ```json
  {
    "code": 0,        // 0=成功，非0=错误码
    "msg": "success", // 错误描述
    "data": {}        // 业务数据
  }
  ```
- **认证方式**：JWT Token（Header: `Authorization: Bearer {token}`）
- **分页参数**：`pageNum`（页码，从1开始），`pageSize`（每页条数）
- **分页响应**：
  ```json
  {
    "code": 0,
    "data": {
      "list": [],
      "total": 100,
      "pageNum": 1,
      "pageSize": 20
    }
  }
  ```

## 四、外部API对接

### 4.1 云途安API（新供应商）

**基础信息**：
- 基础URL：`https://ss.yuntinan.com/prod-api`
- 认证方式：JWT Token
- 登录接口：`POST /login` → 获取token

**核心接口**：

| 接口 | 方法 | 路径 | 用途 |
|------|------|------|------|
| 登录 | POST | `/login` | 获取JWT Token |
| 面积统计 | GET | `/machine/acre/bar` | 获取各类作业面积汇总 |
| 设备列表 | POST | `/machine/carinfo/list` | 获取所有设备（注意：有10条限制） |
| 设备详情 | GET | `/machine/carinfo/{id}` | 单设备详细信息 |
| 作业记录 | POST | `/machine/acre/listp` | 获取设备作业记录（含面积、围栏） |
| 实时监控 | POST | `/machine/monitor/list` | 获取设备实时位置 |
| 作业类型 | GET | `/machine/calendar/list/newListWorkType` | 获取作业类型配置 |

**特殊处理**：
- 设备列表接口有硬限制（最多返回10条），需要逐个设备号查询
- 作业类型编码映射：1=旋耕, 2=深翻, 3=秸秆还田, 23=抛洒

### 4.2 旧供应商API（北斗农机）

**基础信息**：
- 基础URL：`http://60.188.243.23:28111`
- 认证方式：Header签名 `sign = MD5(毫秒时间戳 + key)`
- 账号：user=`2dd7d00ac0ac421caa398f47c65cee6b`, key=`998c50f71d4740a79d11d0101f196f8f`

**核心接口**：

| 接口 | 方法 | 路径 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 轨迹查询 | POST | `/machine/gps/query` | macid, day, startTime, endTime | GPS点列表 |
| 实时位置 | POST | `/machine/gps/location` | macid | 当前位置 |
| 面积查询 | POST | `/machine/gps/area` | macid, day | 面积（字符串，如"2.6793"） |

**限制说明**：
- 面积接口不区分作业类型
- 无达标面积、无地块围栏数据
- 需要按设备逐个查询，建议并发控制（5个并发）

## 五、核心业务逻辑

### 5.1 数据同步流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  定时触发    │────►│  获取Token   │────►│  拉取设备列表 │────►│  逐个查询    │
│  (每天2点)   │     │             │     │             │     │  作业数据    │
└─────────────┘     └─────────────┘     └─────────────┘     └──────┬──────┘
                                                                    │
┌─────────────┐     ┌─────────────┐     ┌─────────────┐            │
│  记录同步日志 │◄────│  批量写入DB  │◄────│  数据转换    │◄───────────┘
│             │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘
```

**关键逻辑**：
1. **全量同步**：首次或数据异常时，拉取所有历史数据
2. **增量同步**：日常定时任务，只拉取最近7天数据
3. **去重策略**：`t_number + work_date + work_type_name + source` 唯一索引
4. **并发控制**：旧供应商API限制，最多5个并发请求

### 5.2 面积计算与确认流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  GPS轨迹上传 │────►│  系统计算面积 │────►│  农户查看确认 │────►│  补贴核算    │
│  (设备自动)  │     │  (ok_acre)   │     │  (手机H5)   │     │  (农业局)   │
└─────────────┘     └─────────────┘     └──────┬──────┘     └─────────────┘
                                               │
                                               ▼
                                        ┌─────────────┐
                                        │  有异议申诉  │
                                        │  (人工处理)  │
                                        └─────────────┘
```

**关键字段**：
- `acre`：API返回的原始作业面积
- `ok_acre`：达标面积（用于补贴计算，可能等于或小于acre）
- `confirm_status`：0=未确认，1=已确认，2=有异议
- `subsidy_rate`：补贴标准（元/亩），从work_stage_types表读取
- `subsidy_amount`：补贴金额 = ok_acre × subsidy_rate

### 5.3 农户端权限设计

- 农户登录后，只能查看关联合作社的设备作业记录
- 一个农户可以关联多个合作社
- 农户确认操作记录日志，便于追溯

## 六、已知问题与规避方案

### 6.1 Node.js版本已修复的问题

| 问题 | 原因 | 修复方案 |
|------|------|----------|
| device_mapping表不存在 | 设计变更后未清理 | 已移除所有对该表的引用 |
| 同步一次性处理3000条超时 | 无分批处理 | 已改为每批200条 |
| 数据库连接未释放 | 缺少finally | 已添加try...finally |

### 6.2 需要Java版特别注意的

| 问题 | 建议方案 |
|------|----------|
| 云途安API分页限制 | 设备列表只能返回10条，需要按设备号逐个查询 |
| 旧供应商API并发限制 | 建议5个并发，每请求间隔200ms |
| 数据去重 | 使用数据库唯一索引，捕获DuplicateKeyException |
| 定时任务重叠 | 使用分布式锁（Redis或数据库锁） |
| 敏感信息加密 | 密码使用bcrypt，API密钥不要硬编码 |

```yaml

server:
  port: 8080

spring:

    base-url: http://60.188.243.23:28111

  secret: ${JWT_SECRET:your-secret-key-change-in-production}
```

1. 启动MySQL、Redis
2. 执行数据库初始化脚本（flyway或手动执行schema.sql）
3. 启动Java应用
4. 访问 `http://localhost:8080/health` 验证健康状态
5. 执行首次数据同步
