# 旧供应商同步问题排查与修复报告

**问题发现时间**: 2026-04-18  
**问题描述**: 服务器上没有更新旧设备的数据  
**修复状态**: ✅ 已修复

---

## 🔍 问题分析

### 根本原因

定时任务代码中引用的CSV文件路径错误，导致无法导入设备映射表。

**错误的路径**:
```javascript
// services/scheduler.js 第185行
const csvPath = path.join(__dirname, '..', '终端映射表_清理版.csv');

// routes/oldSupplier.js 第73行
const csvPath = path.join(__dirname, '..', '终端映射表_清理版.csv');
```

**实际文件位置**:
```
templates/旧供应商终端映射表.csv
```

**历史文件位置**（已被移动到history目录）:
```
history/终端映射表_清理版.csv
```

---

## 📋 问题影响

由于CSV文件路径错误，导致以下问题：

1. **定时任务执行失败**: 凌晨4点的定时任务尝试导入不存在的CSV文件
2. **设备表为空**: `old_supplier_devices` 表中没有设备数据
3. **同步无数据**: `syncDayData()` 方法检测到设备数为0，直接返回
4. **前端无显示**: 页面上"旧供应商"区块不显示任何设备

## 🔧 其他可能的问题

如果修复后仍然没有数据，请检查以下方面：

### 1. 环境变量未生效

**检查**:
```bash
cat .env | grep ENABLE_OLD_SYNC
```

**应该看到**:
```
ENABLE_OLD_SYNC=true
```

**如果没有或为false**:
```bash
echo "ENABLE_OLD_SYNC=true" >> .env
pm2 restart agri-gateway
```

### 2. 设备表为空

**检查**:
```sql
SELECT COUNT(*) FROM old_supplier_devices;
```

**如果为0**:
```bash
node import-old-devices-from-csv.js
```

### 3. API无法访问

**测试**:
```bash
curl -X POST http://60.188.243.23:28111/api/area \
  -H "Content-Type: application/json" \
  -d '{"macid":"test","day":"2026-04-17"}'
```

**如果超时或失败**:
- 检查服务器网络
- 检查防火墙规则
- 联系API提供方

### 4. 所有设备当天作业面积为0

这是正常情况，可能是：
- 设备当天确实没有作业
- API返回的数据为0

**验证**:
```sql
SELECT work_date, COUNT(*), SUM(acre) 
FROM work_records 
WHERE source IN ('old', 'old_api')
GROUP BY work_date
ORDER BY work_date DESC
LIMIT 7;
```

## 💡 预防措施

### 1. 添加文件存在性检查

在 `scheduler.js` 中添加：
```javascript
if (!fs.existsSync(csvPath)) {
  console.error(`[错误] CSV文件不存在: ${csvPath}`);
  console.error('[提示] 请确保 templates/旧供应商终端映射表.csv 存在');
  return;
}
```

### 2. 添加定时任务执行日志

创建独立的日志文件记录定时任务执行情况：
```javascript
const fs = require('fs');
const logFile = 'logs/old-sync.log';
fs.appendFileSync(logFile, `${new Date().toISOString()} - 同步开始\n`);
```

### 3. 添加健康检查接口

增加一个API端点检查旧供应商同步状态：
```javascript
router.get('/old-sync-status', async (req, res) => {
  const deviceCount = await db.queryOne('SELECT COUNT(*) as total FROM old_supplier_devices');
  const todayRecords = await db.queryOne(
    `SELECT COUNT(*) as count FROM work_records 
     WHERE source IN ('old', 'old_api') 
     AND work_date = CURDATE()`
  );
  
  res.json({
    devices: deviceCount.total,
    todayRecords: todayRecords.count,
    lastSync: '...' // 从日志读取
  });
});
```
