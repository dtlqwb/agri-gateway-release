## 🔧 部署步骤

### 1️⃣ 上传文件
使用 FTP/SFTP 或 scp 上传上述2个文件到服务器对应目录

### 2️⃣ 重启服务
```bash
pm2 restart agri-gateway
```

### 3️⃣ 查看日志
```bash
pm2 logs agri-gateway --lines 50
```

**关键日志**（确认修复生效）:
```
[DB] 修复设备数据: 新增 20 台，更新 0 台
[DB] 修复 acre 数据: X 条记录已回填
```

### 4️⃣ 运行诊断
```bash
cd /path/to/agri-gateway-release
node diagnose-data-consistency.js
```

**预期输出**:
```
=== work_records vs machines 一致性检查 ===
work_records 旧供应商设备数: 38
machines 旧供应商设备数: 38
差异: 0 ✅

work_records 旧供应商总面积: 9884.61
machines 旧供应商总面积: 9884.61
差异: 0 ✅
```

### 5️⃣ 前端验证
访问网页，检查统计数据：

| 统计项 | 目标值 | 实际值 | 状态 |
|--------|--------|--------|------|
| 今年作业总面积 | 15686.26亩 | _____ | ⬜ |
| 耕的面积 | 11828.53亩 | _____ | ⬜ |

| 旧供应商设备数 | ~38台 | _____ | ⬜ |

---

## ⚠️ 注意事项

1. **首次重启会自动执行修复**，可能需要几秒钟
2. 如果日志显示"新增 0 台"，说明设备表已经是最新的
3. 如果诊断脚本显示有差异，请检查是否有新数据正在同步
4. 修复后应该**零偏差**，所有数据与旧供应商网页完全一致

```bash

```

### 问题2: 诊断脚本报错

```bash

```
